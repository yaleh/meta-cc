什么是 BAIME？

BAIME (Bootstrapped AI Methodology Engineering) 是一个统一框架，整合了三个针对 LLM 开发优化的互补方法论：

🔄
OCA 循环

观察-编码-自动化的核心迭代框架

🔬
实证验证

科学方法和数据驱动决策

📊
价值优化

双层价值函数进行定量评估

💡 核心创新

BAIME 将方法论开发视为软件开发——通过实证观察、自动化测试、持续迭代和定量指标来进行。

OCA 循环：核心流程

观察 → 编码 → 自动化 → 演化

观察 (Observe)
→
编码 (Codify)
→
自动化 (Automate)
→
演化 (Evolve)

每个阶段都有明确的目标和产出，通过迭代不断优化

🔍 观察阶段

目标：收集当前实践的实证数据

  • 使用 meta-cc 工具分析会话历史
  • Git 分析提交模式
  • 代码指标（覆盖率、复杂度）

📝 编码阶段

目标：提取模式并文档化方法论

  • 从数据中识别模式
  • 形成假设
  • 创建 Markdown 文档

⚙️ 自动化阶段

目标：将方法论转换为自动化检查

  • 检测：识别模式适用时机
  • 验证：检查合规性
  • 执行：CI/CD 门禁

🔄 演化阶段

目标：持续改进方法论

  • 对开发过程使用工具
  • 发现元模式
  • 优化方法论

双层价值函数

每次迭代计算两个独立的质量分数，确保既完成任务又构建可复用方法论：

📈 V_instance：实例层质量

衡量具体交付物的质量

示例：

  • 测试方法论：覆盖率 × 质量 × 稳定性 × 性能
  • CI/CD 流水线：速度 × 可靠性 × 自动化 × 可观测性

目标：≥ 0.80

🎯 V_meta：方法论质量

衡量方法论的可复用性

组成部分：

  • 完整性：方法论是否完整文档化
  • 有效性：提供多少倍速度提升
  • 可复用性：跨项目可迁移百分比
  • 验证性：是否经过实证验证

目标：≥ 0.80

✅ 收敛标准

只有当 两个层面都达到 ≥ 0.80 时，方法论才算完成和就绪。如果只有一个收敛，需要继续迭代。

快速开始

3 步体验 BAIME

最简单的方式：只需给 Claude 双重目标（构建方法论 + 完成任务）

1 明确说明你的目标

使用 BAIME 构建 [领域] 能力（元目标）并完成以下任务（实例目标）：
- [具体任务 1]
- [具体任务 2]

2 Claude 自动执行

  • 使用 iteration-prompt-designer 创建实验结构
  • 使用 iteration-executor 运行迭代（0 → 1 → 2 → ... → N）
  • 使用 knowledge-extractor 创建可复用技能
  • 完成你的具体任务

3 获得产出

  • ✅ 任务完成（README 更新、文档创建等）
  • ✅ 可复用方法论（技能包）
  • ✅ 完整实验轨迹（模板、模式、工具）

💡 核心洞察

你不需要理解 BAIME 就能使用它 —— 只需陈述：

  1. 元目标：你想构建什么能力
  2. 实例目标：你需要完成什么任务

Claude 会自动处理 BAIME 框架、专用代理和迭代管理。

专用代理

BAIME 提供三个专用 Claude Code 子代理简化实验执行：

📝
iteration-prompt-designer

何时使用：实验开始时

作用：为你的实验设计全面的 ITERATION-PROMPTS.md

节省：2-3 小时设置时间

⚙️
iteration-executor

何时使用：每次迭代（0, 1, 2, ...）

作用：通过完整的 OCA 循环执行迭代

优势：一致的结构 + 系统化价值计算

💎
knowledge-extractor

何时使用：实验收敛后

作用：提取知识并转换为可复用方法论

节省：4-6 小时知识整理时间
